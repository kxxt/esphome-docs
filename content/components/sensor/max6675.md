The `max6675`   temperature sensor allows you to use your MAX6675 thermocouple
temperature sensor ([datasheet](https://datasheets.maximintegrated.com/en/ds/MAX6675.pdf), `SainSmart`_) with ESPHome

{{< img src="max6675-full.jpg" alt="Image" caption="MAX6675 K-Type Thermocouple Temperature Sensor." width="60.0%" class="align-center" >}}

As the communication with the MAX6675 is done using SPI, you need
to have an [SPI bus](#spi) in your configuration with the **miso_pin** set (MOSI is not required).

Connect `GND`   to `GND`  , `VCC`   to `3.3V`   and the other three `MISO`   (or `SO`   for short),
`CS`   and `CLOCK`   (or `CLK`  ) to free GPIO pins.

{{< img src="temperature.png" alt="Image" width="80.0%" class="align-center" >}}

```yaml
# Example configuration entry
spi:
  miso_pin: D0
  clk_pin: D1

sensor:
  - platform: max6675
    name: "Living Room Temperature"
    cs_pin: D2
    update_interval: 60s

```
## Configuration variables:

- **cs_pin** (**Required**, [Pin Schema](#config-pin_schema)): The Chip Select pin of the SPI interface.
- **update_interval** (*Optional*, [Time](#config-time)): The interval to check the sensor. Defaults to `60s`  .

- **spi_id** (*Optional*, [ID](#config-id)): Manually specify the ID of the [SPI Component](#spi) if you want
  to use multiple SPI buses.
- All other options from [Sensor](#config-sensor).
